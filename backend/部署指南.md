# 🌈 彩虹创口贴 - 后端部署指南

## 📋 概述

本文档介绍如何将"彩虹创口贴"应用的后端服务部署到云服务器，实现情绪数据的云端存储。

## 🗄️ 数据存储说明

### 本地存储（开发环境）
- **前端**: 使用 `localStorage` 通过 Zustand 的 `persist` 中间件存储
- **后端**: 使用 SQLite 数据库 (`emotion_data.db`)

### 云端存储（生产环境）
- 部署到云服务器后，所有情绪记录将保存到服务器数据库
- 前端会自动将每日情绪记录同步到后端

## 🚀 部署方式

### 方式一：魔塔创空间（推荐）

1. **登录魔塔创空间**: https://www.modelscope.cn/my/minds

2. **创建新创空间**
   - 选择"FastAPI"模板
   - 选择 Python 3.10+ 环境

3. **上传代码**
   ```bash
   # 将以下文件上传到创空间
   - app.py              # 主入口文件
   - requirements.txt    # Python 依赖
   - client/dist/        # 构建后的前端静态文件
   ```

4. **配置环境变量**
   ```
   QWEN_API_KEY=your_api_key_here
   QWEN_API_URL=https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
   SERVER_PORT=7860
   ```

5. **启动服务**
   - 创空间会自动运行 `start.bat` 或 `python app.py`
   - 服务启动后会自动创建 SQLite 数据库

6. **访问应用**
   - 通过创空间提供的公网 URL 访问

### 方式二：自有云服务器（如阿里云、腾讯云）

#### 1. 服务器准备

```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装 Python 3.10+
sudo apt install python3.10 python3-pip -y

# 安装 Node.js（用于构建前端）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y
```

#### 2. 上传代码

```bash
# 创建应用目录
mkdir -p /opt/rainbow-helper
cd /opt/rainbow-helper

# 上传项目文件（使用 scp 或 git）
scp -r * user@server:/opt/rainbow-helper/
```

#### 3. 安装依赖

```bash
# 安装 Python 依赖
pip3 install -r requirements.txt

# 安装前端依赖并构建
cd client
npm install
npm run build
cd ..

# 将构建产物移动到 static 目录
mkdir -p static
cp -r client/dist/* static/
```

#### 4. 配置环境变量

创建 `.env` 文件：
```bash
nano .env
```

内容：
```env
QWEN_API_KEY=your_api_key_here
QWEN_API_URL=https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
SERVER_PORT=7860
```

#### 5. 使用 systemd 管理服务

创建服务文件：
```bash
sudo nano /etc/systemd/system/rainbow-helper.service
```

内容：
```ini
[Unit]
Description=Rainbow Helper Backend Service
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/rainbow-helper
Environment="PATH=/usr/bin:/usr/local/bin"
ExecStart=/usr/bin/python3 /opt/rainbow-helper/app.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable rainbow-helper
sudo systemctl start rainbow-helper
sudo systemctl status rainbow-helper
```

#### 6. 配置 Nginx 反向代理（可选）

```bash
sudo apt install nginx -y
sudo nano /etc/nginx/sites-available/rainbow-helper
```

Nginx 配置：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:7860;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件直接由 FastAPI 处理
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/rainbow-helper /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

#### 7. 配置防火墙

```bash
# 如果使用 UFW
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 如果使用云服务器，在控制台安全组开放端口
```

### 方式三：Docker 部署

1. **创建 Dockerfile**（已包含在项目中）

```dockerfile
FROM python:3.10-slim

WORKDIR /app

# 安装 Node.js
RUN apt-get update && apt-get install -y \
    curl \
    && curl -fsSL https://deb.nodesource.com/setup_20.x | bash - \
    && apt-get install -y nodejs \
    && rm -rf /var/lib/apt/lists/*

# 安装 Python 依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 构建前端
WORKDIR /app/client
RUN npm install && npm run build

# 移动静态文件
WORKDIR /app
RUN mkdir -p static && cp -r client/dist/* static/

# 暴露端口
EXPOSE 7860

# 启动命令
CMD ["python", "app.py"]
```

2. **构建镜像**

```bash
docker build -t rainbow-helper .
```

3. **运行容器**

```bash
docker run -d \
  --name rainbow-helper \
  -p 7860:7860 \
  -v /opt/rainbow-data:/app/data \
  -e QWEN_API_KEY=your_api_key_here \
  rainbow-helper
```

## 📊 API 端点说明

### 情绪记录相关

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/emotions?year=2026&month=3` | GET | 获取指定月份的情绪记录 |
| `/api/emotions` | POST | 保存单条情绪记录 |
| `/api/emotions/batch` | POST | 批量保存情绪记录 |
| `/api/emotions` | DELETE | 删除所有情绪记录 |
| `/api/emotions/all` | GET | 获取所有情绪记录（调试用） |

### 聊天相关

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/rainbow-chat/greeting` | GET | 获取问候语 |
| `/api/rainbow-chat/rainbow-chat` | POST | AI 聊天接口 |

## 🔧 故障排查

### 查看日志

```bash
# systemd 服务日志
sudo journalctl -u rainbow-helper -f

# Nginx 日志
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

### 数据库检查

```bash
# 检查数据库文件是否存在
ls -la /opt/rainbow-helper/emotion_data.db

# 使用 sqlite3 查看数据
sqlite3 /opt/rainbow-helper/emotion_data.db
SELECT * FROM emotion_records ORDER BY date DESC LIMIT 10;
```

### API 测试

```bash
# 测试健康检查
curl http://localhost:7860/health

# 测试获取情绪记录
curl "http://localhost:7860/api/emotions?year=2026&month=3"

curl -X POST http://localhost:7860/api/emotions \
  -H "Content-Type: application/json" \
  -d '{"record":{"id":"test-123","date":"2026-03-11","emotion":"happy","intensity":4}}'
```

## 📱 前端配置

### 开发环境
前端会自动使用当前服务器的 API，无需额外配置。

### 生产环境
如果需要指定 API 地址，可以在构建时设置环境变量：

```bash
# 在 client/.env 文件中设置
VITE_API_BASE_URL=https://your-domain.com
```

然后重新构建前端：
```bash
cd client
npm run build
```

## 🔐 安全建议

1. **使用 HTTPS**: 通过 Let's Encrypt 配置免费 SSL 证书
2. **限制 CORS**: 在生产环境中限制允许的域名
3. **API 认证**: 为敏感操作添加 token 验证
4. **定期备份**: 备份 `emotion_data.db` 数据库文件
5. **监控日志**: 设置日志监控和告警
